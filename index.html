Floating Tower of Hanoi, 2017, Worcester Polytechnic Institute, MA link
I built this game with my friends in my first hackathon. During the hackathon, we were introduced to leap motion and found it fascinating, so we decided to build a game with it and choose the game floating tower of hanoi as the topic. Since my team all played this game in our childhood time, we designed it with lots of colors and pretty cute BGM.
We mainly used three hand interactions: pick, point and release. While play's hand moves to one of the three sections, the current section will be highlighed and the other two will be blured.
Path-finding app for Brigham and Women Hospital, 2018, Worcester Polytechnic Institute, MA link
The goal was to create an interface that allowed patients to find the easiest path to their destination, a database that records all locations and paths information, and a program that connects interface and database together.
As the team lead for the ten-person team, I hosted daily scrum meetings, delegated tasks, initiated progress check-ins, and provided support in designing inputs and outputs. We built the application with java, javaDb and javafx. In the application, we implemented a strategy pattern that chose among A*, Dijkstra, BFS and DFS algorithms. The program applied singleton pattern and restricted instantiation of classes to one instance. After finishing the database and program logic, we started to beatify the UI with hiding most the control panels and showing the map as 90% of the UI.
PositiveKitty, 2020, Worcester Polytechnic Institute Hackathon, MA link
Inspiration: A productive work space consists of a positive atmosphere--even when objective statements and criticisms are shared, they can be done constructively. Aggressive conversations are also not tolerable. We wanted to improve the atmosphere of the Slack workspace by doing something when negative sentiment is detected.
What it does: It uses AWS Comprehend to analyze the conversation in each channel. When the sentiment value remain Negative, the bot sends cat gifs to the user that made the negative statement in order to remind the user to stay positive and potentially disrupt the flow of negative energy.
How we built it: Slack hook per message → AWS API Gateway → AWS Lambda → AWS Comprehend → AWS Dynamo DB for saving historical sentiment scores → detect negativity in the lambda → slack query giphy cat → slack bot send cat gif to user
Classroom Labeling System, 2018, Worcester Polytechnic Institute, MA link
Inspiration: Labelling training data for facial recognition by hand is tedious and inefficient. this project was to create an application to simplify this labelling process. By creating an application that allows users to view, edit, and delete face annotations for videos, the process of creating training data for face detection in videos is simplified greatly.
What it can do: This applicaiton allows students from Machine Learning group to login into the system and check their assigned videos. Users can also label people in the video and able to drag the label to change its shape and position. Any updates from user would be automatically updated to the cloud database so that user can work on labeling from anypoint they left last time.
